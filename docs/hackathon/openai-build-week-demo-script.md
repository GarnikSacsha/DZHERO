# DZHERO Agent Studio — demo script under three minutes

Last updated: **2026-07-17**

## Before recording

- Use the deployed app at `https://openaibuildweek.up.railway.app` in a signed-out/private browser session.
- Enter a dedicated English coffee-shop demo workspace with a complete Brand Brain.
- Prepare one verified public YouTube or TikTok URL and one backup source.
- Keep a completed rehearsal run available in case provider latency would exceed the video limit.
- Clear or label old Content Plan items so the seven approved additions are obvious.
- Do not show keys, Railway variables, personal billing, private customer data, or reusable passwords.
- Do not describe a deterministic fixture, cached result, or saved output as a fresh provider run.

## Narration and clicks

### 0:00–0:18 — The problem

Open **Agent Studio · Beta** from the existing DZHERO workspace.

**Say:** “A small business owner does not need another blank AI chat. They need a grounded decision: what to shoot, why it fits the brand, and what the team should publish next.”

### 0:18–0:38 — Signals and two entry modes

Briefly point to **Signals** and the separately labelled **Find fresh signals** action, then return to Agent Studio.

**Say:** “DZHERO can pull a budget-bounded mix from brand accounts, keywords, hashtags, and trends, or work from a specific public Reel. Inside Agent Studio, Choose from my Signals selects from the workspace bank, while Adapt a Reel starts from one chosen source.”

Use **Adapt a Reel** for the complete story.

### 0:38–0:58 — Start a real run

Choose **New run** if needed, paste the verified public source, and use:

> Bring more weekday morning visits to a neighborhood coffee shop with a low-budget Reel one person can shoot.

Click **Start agent team**.

**Say:** “Jeryk manages a bounded backend workflow. The progress rail reflects persisted server stages, not a cosmetic timer.”

### 0:58–1:27 — GPT-5.6 orchestration and evidence

Show the specialists and open **Grounded evidence** plus the safe activity trace.

**Say:** “GPT-5.6 agents select the mechanic, adapt it to the brand, produce, critique, plan, and manage the package. Gemini has one narrow job: observe the video. Social media is resolved through the source provider, while evidence stays separate from metadata and user notes.”

### 1:27–1:56 — Production quality

Show the full hero, first-two-second hook, shot-by-shot scenes, production notes, evidence references, and quality scores.

**Say:** “The output must be shoot-ready: concrete scenes, tension, proof, CTA, and production notes for one person with a phone. An independent Critic uses stable revision requirements and gets one bounded correction pass. If quality still fails, the run stops honestly.”

### 1:56–2:24 — Human-directed Hybrid

Select two directions and click **Combine two directions**, then create the Hybrid. Cut to the completed rehearsal result if live latency would exceed the limit.

**Say:** “Compact alternatives are directions, not fake finished scripts. The owner combines two ideas through a real additional GPT-5.6 pass. Critic checks the result again, and Content Planner rebuilds the week around it.”

### 2:24–2:47 — One signal to one week

Scroll through the seven-day plan.

**Say:** “This is not seven paraphrases of one Reel. The approved mechanic becomes a connected week across Reels, Stories, and posts, each with a distinct objective, hook, and CTA.”

### 2:47–3:00 — Approval and Codex

Approve the package, open Content Plan, and finish on the seven items.

**Say:** “Only the human approves the workspace write. Codex helped us turn this product decision into tested agent contracts, recovery paths, UI, deployment, and verification. One grounded signal is now one actionable week.”

## Live recovery

If the public source is blocked or slow:

1. Show the classified source error or context state briefly.
2. Use **Retry automatic video fetch** once when appropriate.
3. Switch to the prepared backup public URL.
4. If providers remain unavailable, show the saved rehearsal run and state that it is a previously completed result.
5. Never imply that metadata-only output is a fresh video observation.

## Final recording checklist

- [ ] Public YouTube URL.
- [ ] Total duration no longer than three minutes.
- [ ] Continuous voiceover or clearly audible narration.
- [ ] Voiceover explains the product, Codex collaboration, and GPT-5.6 runtime use.
- [ ] English UI and English generated package throughout.
- [ ] Final human approval and seven Content Plan items are visible.
- [ ] No credentials, personal billing, private data, or unlicensed music.
- [ ] Captions are enabled or burned in.
