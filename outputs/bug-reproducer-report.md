## Root cause

The finalize route rethrew provider usage denials and the persistence boundary enforced trial/save limits even for the mandatory first Brand Brain completion.

## Approved fix

For incomplete Brand Brain only, detect unavailable AI before provider work, use the deterministic fallback, and allow the mandatory first save when trial/save enforcement would otherwise trap the user. Completed-brand Settings edits remain strictly metered.

**Why this is causal:** The change affects only the two blocking boundaries reached by first-time finalization; the same request now persists without provider calls while the existing completed-brand denial regression still returns 402.
